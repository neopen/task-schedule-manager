核心特性

NeoTask 提供了一套完整的企业级任务调度解决方案

高性能

基于异步IO实现，支持每秒处理数万级任务，延迟低于10毫秒

分布式

支持 Redis 作为后端存储，实现多节点分布式部署和任务调度

定时任务

支持延时执行、周期执行和 Cron 表达式，满足各种定时需求

监控告警

内置指标收集和健康检查，支持自定义告警规则

分布式锁

基于 Redis 的分布式锁，确保任务在集群中只被一个节点执行

多种存储

支持 Memory、SQLite、Redis 三种存储后端，按需选择

自动重试

任务失败自动重试，支持配置重试次数和退避策略

可观测性

事件总线 + 指标收集，支持 Prometheus 集成

快速上手

5 分钟快速入门 NeoTask 任务调度系统

# 安装
pip install neotask          # 基础安装（内存存储）
pip install neotask[sqlite]  # 带 SQLite 持久化
pip install neotask[redis]   # 带 Redis 分布式支持

# 基础使用
from neotask import TaskPool

async def process_image(data: dict) -> dict:
    """处理图片任务"""
    print(f"开始处理图片: {data['url']}")
    await asyncio.sleep(1)
    return {"status": "success", "url": data['url']}

# 创建任务池
pool = TaskPool(executor=process_image)

# 提交任务
task_id = pool.submit({"url": "https://example.com/image.jpg"})

# 等待结果
result = pool.wait_for_result(task_id, timeout=30)
print(result)

# 定时任务
from neotask import TaskScheduler

scheduler = TaskScheduler(executor=process_image)

# 延时 60 秒执行
scheduler.submit_delayed({"action": "reminder"}, delay_seconds=60)

# 每 5 分钟执行一次
scheduler.submit_interval({"action": "heartbeat"}, interval_seconds=300)

# 每天 9 点执行
scheduler.submit_cron({"action": "daily"}, cron_expr="0 9 * * *")
