配置说明

完整的 NeoTask 配置参数详解。

配置概览

NeoTask 提供灵活的配置方式，支持代码配置、环境变量和默认值三级优先级。

from neotask import TaskPool, TaskPoolConfig

config = TaskPoolConfig(
    storage_type="redis",
    redis_url="redis://localhost:6379",
    worker_concurrency=20,
    max_retries=3,
    enable_metrics=True
)

pool = TaskPool(executor=my_task, config=config)

存储配置详解

内存存储（开发测试）

config = TaskPoolConfig(storage_type="memory")

特点： 速度快，无 I/O 开销；进程重启后任务丢失。
适用场景： 本地开发调试、单元测试、临时任务。

SQLite 存储（单机生产）

config = TaskPoolConfig(
    storage_type="sqlite",
    sqlite_path="/data/neotask.db"
)

特点： 持久化存储，重启不丢失；无需额外服务；不支持多节点共享。
适用场景： 单机生产部署、需要任务持久化、中小规模任务（< 10万/天）。

Redis 存储（分布式）

config = TaskPoolConfig(
    storage_type="redis",
    redis_url="redis://localhost:6379/0",
    lock_type="redis",
    lock_timeout=30
)

Redis URL 格式：

# 基础格式
redis://localhost:6379/0

# 带密码
redis://:password@localhost:6379/0

# SSL 连接
rediss://localhost:6379/0

特点： 多节点共享任务；支持分布式部署；高性能。
适用场景： 多节点分布式部署、高并发任务处理。

支持通过环境变量配置，优先级低于代码配置。

环境变量对应配置默认值

配置调优建议

Worker 并发数

任务类型推荐值说明
CPU 密集型CPU 核心数避免上下文切换
IO 密集型CPU 核心数 × 2-4提高并发
混合型CPU 核心数 × 2平衡性能

预取大小

场景推荐值说明
低延迟要求1-5任务快速响应
高吞吐要求20-50减少网络往返
任务耗时较长10-20平衡内存和吞吐

队列大小

场景推荐值说明
内存有限1000-5000控制内存使用
标准配置10000-50000平衡性能
高吞吐100000+大量任务堆积

超时设置

参数推荐值说明
task_timeout平均耗时 × 3避免误判超时
lock_timeout任务耗时 × 2避免锁提前释放
wait_timeout任务耗时 × 2等待结果超时

配置验证

def validate_config(config: TaskPoolConfig) -> bool:
    if config.storage_type not in ["memory", "sqlite", "redis"]:
        raise ValueError(f"Invalid storage_type: {config.storage_type}")
    if config.worker_concurrency < 1:
        raise ValueError("worker_concurrency must be >= 1")
    if config.max_retries < 0:
        raise ValueError("max_retries must be >= 0")
    if config.queue_max_size < 1:
        raise ValueError("queue_max_size must be >= 1")
    return True
