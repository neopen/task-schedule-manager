快速上手

5 分钟快速入门 NeoTask 任务调度系统。

安装

# 基础安装（内存存储，适合开发测试）
pip install neotask

# 带 SQLite 持久化（适合单机生产）
pip install neotask[sqlite]

# 带 Redis 分布式支持（适合多节点部署）
pip install neotask[redis]

# 完整安装（包含所有扩展）
pip install neotask[full]

5 分钟入门

1. 定义任务处理函数

from neotask import TaskPool

async def process_image(data: dict) -> dict:
    """处理图片任务"""
    print(f"开始处理图片: {data['url']}")
    import asyncio
    await asyncio.sleep(1)
    return {
        "status": "success",
        "url": data['url'],
        "size": 1024
    }

2. 创建任务池并提交任务

# 创建任务池
pool = TaskPool(executor=process_image)

# 提交任务
task_id = pool.submit({
    "url": "https://example.com/image.jpg"
})
print(f"任务已提交: {task_id}")

3. 等待并获取结果

# 同步等待（阻塞当前线程）
result = pool.wait_for_result(task_id, timeout=30)
print(f"任务结果: {result}")

# 关闭任务池
pool.shutdown()

4. 异步方式（推荐）

import asyncio

async def main():
    pool = TaskPool(executor=process_image)
    pool.start()
    
    task_id = await pool.submit_async({"url": "https://example.com/image.jpg"})
    result = await pool.wait_for_result_async(task_id)
    print(f"任务结果: {result}")
    
    await pool.shutdown_async()

asyncio.run(main())

5. 使用上下文管理器（最简洁）

with TaskPool(executor=process_image) as pool:
    task_id = pool.submit({"url": "https://example.com/image.jpg"})
    result = pool.wait_for_result(task_id)
    print(result)
# 自动关闭

定时任务快速入门

from neotask import TaskScheduler

async def send_report(data: dict) -> dict:
    """发送报表"""
    print(f"发送报表: {data['type']}")
    return {"status": "sent"}

# 创建调度器
scheduler = TaskScheduler(executor=send_report)

# 延时 60 秒执行
task_id = scheduler.submit_delayed(
    {"type": "daily"},
    delay_seconds=60
)

# 每 5 分钟执行一次
task_id = scheduler.submit_interval(
    {"type": "heartbeat"},
    interval_seconds=300
)

# 每天 9 点执行
task_id = scheduler.submit_cron(
    {"type": "daily_report"},
    cron_expr="0 9 * * *"
)

# 等待结果
result = scheduler.wait_for_result(task_id)

# 关闭调度器
scheduler.shutdown()

优先级任务

from neotask import TaskPriority

# 高优先级任务（数字越小优先级越高）
high_priority_id = pool.submit(
    {"task": "urgent"},
    priority=TaskPriority.HIGH   # 或 priority=1
)

# 普通优先级（默认）
normal_id = pool.submit({"task": "normal"})

# 低优先级
low_id = pool.submit(
    {"task": "background"},
    priority=TaskPriority.LOW    # 或 priority=3
)

优先级说明

优先级值说明
CRITICAL0最高优先级
HIGH1高优先级
NORMAL2普通优先级（默认）
LOW3低优先级

批量提交

# 批量提交任务
tasks = [
    {"id": 1, "url": "image1.jpg"},
    {"id": 2, "url": "image2.jpg"},
    {"id": 3, "url": "image3.jpg"},
]

# 同步批量提交
task_ids = pool.submit_batch(tasks)

# 异步批量提交
task_ids = await pool.submit_batch_async(tasks)

# 等待所有任务完成
results = pool.wait_all(task_ids)
for task_id, result in results.items():
    print(f"{task_id}: {result}")

事件回调

# 定义事件处理器
async def on_created(event):
    print(f"任务创建: {event.task_id}")

async def on_started(event):
    print(f"任务开始: {event.task_id}")

async def on_completed(event):
    print(f"任务完成: {event.task_id}")

# 注册事件回调
pool.on_created(on_created)
pool.on_started(on_started)
pool.on_completed(on_completed)

常见问题

Q: 同步和异步 API 有什么区别？

同步 API（如 submit）会阻塞当前线程直到操作完成；异步 API（如 submit_async）返回 awaitable，需要在 async 函数中使用。

Q: 如何选择存储类型？

  • memory：开发测试，进程重启丢失
  • sqlite：单机生产，持久化存储
  • redis：分布式部署，多节点共享

Q: 任务失败后会自动重试吗？

会。可通过 max_retries 配置重试次数，retry_delay 配置重试延迟。

Q: 如何监控任务执行情况？

使用事件回调监听任务生命周期；调用 get_stats() 获取实时统计；启用 Web UI（规划中）。
