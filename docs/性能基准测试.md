## 性能优化建议

### 1. Worker 数量配置

- **IO 密集型任务**：设置较高的 worker 数量（如 CPU 核心数的 2-4 倍）
- **CPU 密集型任务**：设置接近 CPU 核心数的 worker 数量
- **轻量任务**：根据实际测试结果调整，避免过多 worker 导致调度开销

### 2. 批量操作

在处理大量任务时，使用批量弹出接口减少锁竞争：

```python
tasks = await queue.pop(batch_size=10)  # 批量获取
```

### 3. 存储后端选择

| 存储类型 | 适用场景 |
|----------|----------|
| Memory | 单机、开发环境、高吞吐量场景 |
| SQLite | 轻量级持久化需求 |
| Redis | 分布式部署、高可用需求 |

## 结论

根据实际测试结果，NeoTask 在以下方面表现优秀：

1. **高吞吐量调度**：基础调度可达 **1697 ops/sec**
2. **高效队列操作**：优先级队列出队可达 **312,000+ ops/sec**
3. **批量操作优化**：批量弹出可提升 **2x** 性能
4. **突发负载处理**：500 任务仅需 **0.08s**
5. **高成功率**：所有测试成功率均为 **100%**

建议根据实际业务场景选择合适的配置参数，以达到最佳性能。
